Today we will learn how to use Firebase to create a simple blog post app.

Firebase

We will use the following Builds to implement a simple blog app.

  • Authentication: for authentication, in this article, we will use Google SignIn method.
  • Cloud Firestore Database: for storing post datas.
  • Storage: Optional, if you want to post with cover images.

Create project

Go to Firebase console, create a new project and complete the instructions of steps.

Build

Authentication

Click the Authentication categorized in Build from the left bar drawer, and click Get Started.

Choose a sign in method for the authentication of your project, we will use Google SignIn method in this article.

Firestore Database

Click the Firestore Database categorized in Build from the left bar drawer, and click Create Database.

Why don’t we use Realtime Database instead? Realtime Database stores data into a huge JSON tree, while Firestore Database stores data and organzied them as collections in documents. With this feature, you can search and filter data easier than the former one.

There are two modes to activate the database:

  1. Release mode: The data stored is private, and will decide whether write and read data according to the rules had set.
  2. Debug mode: The data stored is public, and allows to be written and read. By default, you need to update the rules in 30 days in case you need to keep the permission of writing and reading data.

To enable the data always be written and read, i.e. ignoring the date limit, set them to true.

Storage

Click the Storage categorized in Build from the left bar drawer, and click Get Started.

Like the Firestore Database, you will to decide which mode it will activate and the rules it will use.
