正則表達式：學極都唔識

BlueWhaleYT2022 年 05 月 25 日2023 年 10 月 08 日

正則表達式，在編寫程式的時候會用到，但真的很討厭。

正則表達式

正則表達式 (Regular Expressions，簡稱 Regex) 是對字符串操作的一種邏輯公式，就是用事先定義好的一些特定字符、及這些特定字符的組合，組成一個「規則字符串」，這個「規則字符串」用來表達對字符串的一種過濾邏輯。

主要功能如下：

  1. 校驗數據的有效性
  2. 查找符合要求的文本內容
  3. 對文本進行分割或替換等操作

元字符

元字符在正則表達式具有特殊意義，進而實現不同操作。

  1. 特殊單字符
    • .：任意字符（換行符除外）
    • \d：任意數字
    • \D：任意非數字
    • \w：任意字母、數·字·及底線 _
    • \W：任意非字母、數·字·及底線 _
  2. 空白符
    • \r：回車符 ↵
    • \n：換行符
    • \f：分頁符
    • \t：製表符 Tab
    • \s：任意空白符
    • \S：任意非空白符
  3. 範圍
    • |：或，例如 ab|bc 就是 ab 或 bc
    • [...]：多選一，比對中括號內任一元素
    • [a-z]：比對 a 至 z 之間任一元素
    • [^...]：取反，不會比對中括號內任一元素
    • ()：子表達式，在表達式內嵌套一個子表達式
  4. 量詞
    • *：0 至以上
    • +：1 至以上
    • ?：0 或 1 次
    • {n}：出現 n 次
    • {n,}：出現至少 n 次
    • {n,m}：出現 n 至 m 次
  5. 斷言
    • \b：單詞的首端及尾端
    • ^：行的首端
    • $：行的尾端

特殊單字符

元字符 描述
. 比對除換行符以外的任意字符
\w 比對字母、數字或底線，\W 則相反
\d 比對數字，\D 則相反
\s 比對任意空白符，\S 則相反
. 使用示例

例子 1

比對文本abcde\n
表達式
.
比對數目7
比對結果a、b、c、d、e、\ 、n

例子 2

比對文本abcde\nabc
表達式
ab.
比對數目2
比對結果abc、abc

例子 3

比對文本c:\poloyy
表達式
..\\
比對數目1
比對結果c:\
\w 使用示例

例子 1

比對文本0198aAzZ\w哎_-
表達式
\w
比對數目10
比對結果0、1、9、8、a、A、z、Z、w、_

例子 2

比對文本0198aAzZ\w哎_-
表達式
\w.\w
比對數目3
比對結果019、8aA、Z\w

例子 3

比對文本0198aAzZ\w哎_-
表達式
\w.\\w
比對數目1
比對結果zZ\w
\d 使用示例

例子 1

比對文本123阿a\
表達式
\d
比對數目3
比對結果1、2、3

例子 2

比對文本12a3阿a\
表達式
\d\w\d.
比對數目1
比對結果2a3阿

量詞及限定符

限定符 描述
* 重複 0 次或以上
+ 重複 1 次或以上
? 重複 0 次或 1 次
{n} 重複 n 次
{n,} 重複 n 次或以上
{n,m} 重複 n 至 m 次

限定符可以重複比對

* 使用示例

比對文本yy1112菠蘿

比對文本yy_菠蘿

\d+\w+
比對數目0
比對結果沒有比對結果，因為 \d 沒有至少比對到一個數字

例子 1

比對文本yy12菠蘿

範圍

元字符 描述
| 分支條件，或
[...] 字符集，多選一
[a-z] 指定範圍的字符集，如 a 至 z
[^...] 取反字符集

正則表達式內置已經有比對字母、數字及空白的字符集，我們可以用以下的元字符替代冗長的表達式。

元字符 等價表達式
\w [a-zA-Z0-9_]
\W [^a-zA-Z0-9_]
\s [\n\t\r\f]
\S [^\n\t\r\f]
\d [0-9]
\D [^0-9]

以上就是內置的字符集，但是如果你要比對沒有預定的字符集，就要用 [ ] 手動寫字符集。

例如，寫一個元音字母的字符集，元音字母有 a、e、i、o、u，那就寫 [aeiou]。

字符集只可以比對單一字符，有需要可用限定符進行重複比對。

回溯

在比對過程中，當某一部分的比對失敗後，它會回溯到之前的位置，嘗試其他的比對方式，直到找到合適的比對或所有的選項都嘗試完畢。這個過程可能會消耗較多的時間和資源，尤其是當正則表達式的複雜度很高時。

出現回溯的原因，通常都是因為啟動了貪婪模式。

避免回溯

使用惰性模式或獨佔模式可有效阻止比對過程中出現回溯的情況，兩者都有效阻止，但性質相異。

  • 惰性模式：盡可能比對最少的字符
  • 獨佔模式：與貪婪模式一樣，都是盡可能比對最多字符，但是阻止回溯

模式比對

貪婪模式 (Greedy)

貪婪模式是量詞及限定符特有的特性，貪婪模式，顧名思義就是所比對的次數 「盡可能多」。

惰性 / 非貪婪模式 (Reluctant)

惰性模式，又稱非貪婪模式，就是與貪婪模式相反，所比對的次數 「盡可能少」。

惰性限定符 描述
*? 重複 0 次或以上，但盡可能減少重複
+? 重複 1 次或以上，但盡可能減少重複
?? 重複 0 次或 1 次，但盡可能減少重複
{n,}? 重複 n 次或以上，但盡可能減少重複
{n,m}? 重複 n 至 m 次，但盡可能減少重複

獨佔模式 (Possessive)

與貪婪模式一樣，唯一不同的就是會阻止回溯。

表達式 比對文本 結果
貪婪模式 a{1,3}ab aaab 比對
惰性模式 a{1,3}?ab aaab 比對
獨佔模式 a{1,3}+ab aaab 沒有比對

部分的正則引擎不支援獨佔模式，可用惰性模式取替，但獨佔模式在某些情況下使用，會有較佳效果。

額外補充

比對雙字節字元集

雙字節字元集例如中文、日語及韓語 (CJK) 等利用兩個字節而組成的字元。

  1. 只對正體中文及簡體中文比對
    表達式：[\u4e00-\u9fa5]
  2. 對雙字節字元集比對
    表達式：[^\x00-\xff]

比對泛用格式字串

  • 時間 HH::MM::SS
    ([01]{1}\d|2[0-3]):([0-5]{1}\d):([0-5]{1}\d)
  • 網址 www.example.com
    [w]{3}\.([a-z0-9]+\.)*([a-z]){2,3}
  • 8 位數電話號碼
    [0-9]{8}$

Java 與 JavaScript 的正則表達式使用方法差別

  1. 有兩種創建正則表達式的方法
    顯式宣告：利用 RegExp 物件創建
    var regex = new RegExp("ab+c");
    隱式宣告：利用首尾的 / 作為表達式創建
    var regex = /ab+c/;
  2. 分組是用 ( )
  3. 轉義只用單反斜線 \，如 \d
  1. 創建 Pattern 物件
    Pattern pattern = Pattern.compile("ab+c")
  2. 分組是用 Matcher.group()
  3. 轉義要用雙反斜線 \\，如 \\d
    要轉義 \ 字符，需要加 \\\\
