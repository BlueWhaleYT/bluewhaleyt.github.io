Jetpack Compose 開發 | ViewModel 筆記

BlueWhaleYT2023 年 08 月 29 日2023 年 12 月 02 日

有時候，當我們在自己的應用程式編寫了一些內容，在旋轉螢幕或更換主題模式時，剛寫好的東西就遺失了，這裏難免有些挫敗感，為什麼不寫好自己的程式，有點彆扭就是了。

通常要解決這個問題，基本上就是把你的程式結構進階化，改用 ViewModel。

什麼是 ViewModel？

ViewModel 是 MVVM 架構中最為舉足輕重的角色之一，亦是管理 UI 數據組件的核心用具。

MVVM 是何方神聖

MVVM 是由幾個不同部分組成：

  1. Model：模型，是應用程式的數據模型，處理數據存取
  2. View：視圖：用戶所見的界面，顯示數據和接收用戶輸入
  3. ViewModel：視圖模型，作為綁定模型和視圖的中間層，用於處理用戶界面交互和將數據從模型傳遞到視圖，主要是負責商業邏輯的部分

MVVM 模式的目的是使得開發人員能夠分離用戶界面和業務邏輯，從而使得開發過程更加簡便和靈活。這也方便了測試和維護。

雖然管理 UI 數據是可以直接用 Activity 或 Fragment 來做的，但自 Google 官方推出 ViewModel 至今，強力建議把 UI 數據寫入並存放到 ViewModel。

一般而言，如果你不用 ViewModel，但又想解決界面更新後重建的問題，可能會用：

始終不建議：

  1. 強行封鎖畫面方向：如果你純粹不想用戶旋轉畫面（進而防止資料重設，畫面重建），正常情況下可做這個
    <activity
    android:screenOrientation="portrait"/>
  2. 禁用配置更新
    <activity
    android:configChanges="orientation|screenSize|keyboardHidden"/>
  3. 儲存基礎狀態，用 onSaveInstanceState 和 onRestoreInstanceState
    public class MainActivity extends Activity {
    private static final String SELECTED_ITEM_POSITION = "ItemPosition";
    private int mPosition;
   
    @Override
    protected void onSaveInstanceState(final Bundle outState) {
        super.onSaveInstanceState(outState);
        
        // Save the state of item position
        outState.putInt(SELECTED_ITEM_POSITION, mPosition);   
    }

    @Override
    protected void onRestoreInstanceState(final Bundle savedInstanceState) {
        super.onRestoreInstanceState(savedInstanceState);
        
        // Read the state of item position
        mPosition = savedInstanceState.gettInt(SELECTED_ITEM_POSITION);
    }
}
  4. 用 setRetainInstance()
    public class WorkerFragment extends Fragment {

    // data object we want to retain
    private DataObject data;

    // this method is only called once for this fragment
    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        // retain this fragment
        setRetainInstance(true);
    }

    public void setData(DataObject data) {
        this.data = data;
    }

    public DataObject getData() {
        return data;
    }
}

重點是，以上的方法你應該不會想用了，或者不會有用，現在你寫的是 Compose。

開始編寫

這裏就會記錄一下寫 ViewModel 的程式。

簡單預備一個佈局

setContent {
    AppTheme {
        Surface(
            modifier = Modifier.fillMaxSize(),
            color = MaterialTheme.colorScheme.background
        ) {
            Button(
                onClick = {

                }
            ) {
                Text("Click me to change color")
            }
        }
    }
}

加入 ViewModel

例如叫做 MainViewModel.kt 

class MainViewModel {
    var backgroundColor by mutableStateOf(Color.White) // 預設是白色
        private set

    fun setBackgroundColor() {
        backgroundColor = Color.Red
    }
}

private set 可限利訪問這個屬性的權限，只能在聲明這個屬性的類別可修改，外部則只能讀取。

然後在 MainActivity.kt 宣告這個 ViewModel。

private val viewModel = MainViewModel()

佈局內容修改，更新按鈕點擊事件

Surface(
    modifier = Modifier.fillMaxSize(),
    color = viewModel.backgroundColor
) {
    Button(
        onClick = {
            viewModel.setBackgroundColor()
        }
    ) {
        Text("Click me to change color")
    }
}

你會發現，顏色是有更換的，但是都是依舊旋轉畫面或其他界面更新行為就重建了。

修復 ViewModel

  1. 把 MainViewModel 繼承自 ViewModel
    class MainViewModel : ViewModel() { ... }
  2. 在 MainActivity 修改 viewModel 宣告式
    private val viewModel by viewModels<MainViewModel>()

為 ViewModel 加入參數

一旦加入參數，例如

class MainViewModel(
    str: String
) : ViewModel() { ... }

就會閃退了，報錯：

Cannot create an instance of class com.bluewhaleyt.testcomposeapp.MainViewModel

因為系統再偵測不了 ViewModel。

加入依賴

implementation("androidx.lifecycle:lifecycle-viewmodel-compose:2.6.2")

這個時候就要把 viewModel 挪下來到 AppTheme 裏，加入 factory 參數。

val viewModel = viewModel<MainViewModel>(
    factory = object : ViewModelProvider.Factory {
        override fun <T : ViewModel> create(modelClass: Class<T>): T {
            return MainViewModel(
                str = "Hello World"
            ) as T
        }
    }
)

這個 factory 參數太複雜了，可以加一個複用函數

  1. 寫一個傳入參數 creator 的 factory 構建函數
    inline fun <reified T : ViewModel> setViewModelFactory(crossinline creator: () -> T): ViewModelProvider.Factory {
    return object : ViewModelProvider.Factory {
        @Suppress("UNCHECKED_CAST")
        override fun <T : ViewModel> create(modelClass: Class<T>): T {
            return creator() as T
        }
    }
}
  2. 使用方式
    val viewModel = viewModel<MainViewModel>(
    factory = setViewModelFactory {
        MainViewModel(str = "Hello World")
    }
)
