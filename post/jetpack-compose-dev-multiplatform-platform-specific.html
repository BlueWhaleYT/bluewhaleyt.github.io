Jetpack Compose 開發 | 平台特定程式碼邏輯處理

BlueWhaleYT2023 年 12 月 01 日2023 年 12 月 02 日

在進行開發跨平台的程式，往往需要考慮使用不同平台支援的 API，例如 Android 是用 API A，而 iOS 是用 API B，那在 Compose Multiplatform 要實現出來，就要學會兩種撇步。

平台特定邏輯處理

利用 expect 及 actual

要進行簡單的平台特定邏輯，可以用 expect 和 actual，字面上意思，就是：

  • expect：聲明公共邏輯的期望方法 (abstraction 描象)
  • actual：具體平台實現該方法 (implementation 實現)

獲取平台名稱

官方模板有一個例子：

  1. 在 commonMain 模塊裏有一個 Platform.kt 檔案

    interface Platform {
    val name: String
}

expect fun getPlatform(): Platform

    你會看到，寫了一個新的介面 (interface)，裏面有一個為 name 的變量，大概就是獲取平台的名稱吧。用 expect 寫了一個 getPlatform() 的函數，傳回的類型是 Platform，即剛寫的介面，這個就是公共邏輯，待會就可以讓不同平台使用。

  2. 現在先來看看最熟悉的 Android，在 androidMain 模塊裏的 Platform.android.kt

    import android.os.Build

class AndroidPlatform : Platform {
    override val name: String = "Android ${Build.VERSION.SDK_INT}"
}

actual fun getPlatform(): Platform = AndroidPlatform()

    首先，AndroidPlatform 是繼承自 Platform 介面，裏面的 name 則利用 Build 類的 API 獲取 Android 裝置的名稱及版本，這裏的 actual 寫的 getPlatform() 會與 Platform.kt 寫 expect 的 getPlatform() 產生連結。

  3. 最後，來看看 iOS，同樣地在 iosMain 有一個 Platform.ios.kt

    import platform.UIKit.UIDevice

class IOSPlatform: Platform {
    override val name: String = UIDevice.currentDevice.systemName() + " " + UIDevice.currentDevice.systemVersion
}

actual fun getPlatform(): Platform = IOSPlatform()

    與 Android 的差不多，但用的 API 不同，這裏用的是 UIDevice。

  4. 至於要怎樣把資料顯示出來呢，例如顯示在 Text

    Text(text = getPlatform().name)

expect 及 actual 不屬於 Kotlin 自身語法關鍵字，只是供 Compose Multiplatform 使用的特定語法，expect 和 actual 彼此是產生連結的。

打印日誌 (Log)

  1. commonMain
    interface IALog {
    fun v(tag: String, message: String)
}
expect class IALogImpl(): IALog
  2. androidMain
    import android.util.Log

actual class IALogImpl actual constructor() : IALog {
    override fun v(tag: String, message: String) {
        Log.v(tag, message)
    }
}
  3. iosMain
    import platform.Foundation.NSLog

actual class IALogImpl actual constructor() : IALog {
    override fun v(tag: String, message: String) {
        NSLog("[$tag] $message")
    }
}
