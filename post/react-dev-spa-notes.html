React | JSX 及全面 SPA 單頁應用筆記

BlueWhaleYT2024 年 01 月 08 日2024 年 01 月 08 日

React 是一個用於編寫用戶介面的 JavaScript 函式庫，主要用於構建單頁應用 (SPA) 的網站。

  • 網站部分由 React 管理: 可以讓初學者把 React 程式碼一部分的寫在 HTML 檔案，如將 React 元件嵌入到現有的 HTML 中，增強網站的互動性和功能，而不需要重寫整個網站。
  • 網站全部由 React 管理: 完全使用 React 建構整個網站。這意味著你的網站所有頁面和元件都是使用 React 來開發和渲染。

單頁應用 (SPA) VS 多頁應用 (MPA)

  • 多頁應用: 在絕大多數的網站都是使用多頁應用的，這個網站會有許多個 HTML 頁面，每個頁面都需要從伺服器重新加載。
  • 單頁應用: 在第一次請求時，會向伺服器獲取需要加載的頁面以及資源，當再要進行下一次加載時，不需要再從伺服器加載，而是經由路由動態加載。

React 元件

React 元件封裝了獨立的功能和狀態，並可以接受輸入的屬性，可以被視為獨立的模塊，可以在應用程式中多次使用。每個元件都有自己的生命週期方法和狀態管理，使得開發者可以方便地將 UI 功能分為小塊，並且可以很容易地維護和測試這些元件。

React 元件有兩種函式構建方式：

  1. 類別型元件 (Class-based Component): 需要使用 extends 和 render() 額外語法來渲染 React 介面
  2. 函式型元件 (Functional Component): 不需要寫 render()，編譯後會自動在 return JSX 時叫用 React 提供的函式轉成元件

比較表如下：

函式型元件 類別型元件
語法 定義為 JavaScript 函式 定義為 JavaScript 類別
狀態管理 使用 React Hooks (例如 useState) 使用 this.state 和 this.setState
生命週期方法 使用 React Hooks (例如 useEffect) 有存取生命週期方法 (例如 componentDidMount)
代碼複雜度 語法簡潔、簡明 需要更多樣板代碼
可重用性 容易重用和組 可以重用，但需要更多努力
效能 由於較少的額外開銷，效能稍好 由於額外的開銷，效能稍低
Context API 使用 useContext Hook 使用 this.context
實例變數 不支援實例變數 支援實例變數
代碼組織 函式導向，更容易組織代碼 類別導向，傳統的物件導向程式設計風格
社群偏好 在近年來越來越受歡迎 已成熟且廣泛使用

React 網頁實作

網站部分由 React 管理

只管理網頁內的內容部分，把 React 元件加入到 HTML 網頁中顯示，不需要重寫網站。

加入 React 函式庫：

因為 React 需要使用 JSX (JavaScript XML) 語法，一個把邏輯及變量程式操作加入到 HTML 檔案中，而一般的 JavaScript 是不支援 JSX 的，所以還需要在 Babel 官網安裝 babel 相關套件:

當要在 <script> 標記寫 JSX 程式碼時，就需要加上 text/babel 的 type，註明該程式碼將會使用 Babel 轉譯，在這個 React 環境，就會是把 JSX 語法轉譯為瀏覽器看的懂的 JavaScript 程式碼。

在 HTML 加上一個 <div> 以及賦予一個作為 React 容器的 ID，例如 app。

然後在 <script type='text/babel'> 內加入 JSX 代碼渲染 React 元件至 HTML 網頁。

編寫 JSX 要注意的地方

  • 元素必須只有一個根元素，通常會用 <div> 包起來 這個程式碼是錯誤的，因為它存在了兩個根元素，一個是 <div>，而另一個是 <p>。
  • 定義 class 屬性應改用 className，雖然在 JSX 渲染 React 元件沒有問題，但是當要全面使用 React，即不是單純嵌入 React 元件至 HTML 時候就會報錯，所以最好養成習慣，改為 className
  • JSX 的區域部分，如果要在元件內定義註解 (comment)，可以用一組 { /* */ } 定義註解。另外，在 JavaScript 語法部分可以編寫 JavaScript 程式碼，在 HTML 可以用一組 { } 接收。

{ } 是有直接運行的意思。
